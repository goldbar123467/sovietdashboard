# Pet Dock Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a compact Soviet-themed pet dock to the dashboard right rail that reacts to Codex status and supports lightweight local pet chat.

**Architecture:** Keep pet behavior separate from `CommandBoard.tsx` in a pure `petLogic.ts` module with tests, then render a focused `PetDock.tsx` component. The visual pet is now a local Three.js canvas in `SovietPetScene.tsx` so the dashboard companion can be richer than the packaged Codex app pet format. The pet dock consumes the current Codex dashboard snapshot and emits optional prompt shortcuts back to the existing Agent Comms textarea.

**Tech Stack:** React 19, TypeScript, Tailwind CSS classes, Node test runner with `tsx`, Playwright screenshot verification.

```ts
import assert from "node:assert/strict";
import test from "node:test";
import { petModeForCodex, petReplyForMessage } from "./petLogic";

test("petModeForCodex maps running status to review", () => {
  assert.equal(petModeForCodex({ status: "running", failures: 0, lastRunOk: true }), "review");
});

test("petModeForCodex maps failed latest run to failed", () => {
  assert.equal(petModeForCodex({ status: "idle", failures: 1, lastRunOk: false }), "failed");
});

test("petReplyForMessage answers token questions locally", () => {
  assert.match(petReplyForMessage("why are tokens high?", { totalTokens: 1200, cachedInputTokens: 900 }), /cached/i);
});
```

- [x] **Step 2: Implement logic**

Create deterministic mapping:

- running -> `review`
- latest failed run -> `failed`
- latest successful run -> `salute`
- otherwise -> `idle`

Pet chat stays local and returns short themed strings based on keyword matches.
